AO7: Identification and Authentication Failure

According to Django Docs, "by default, Django uses the PBKDF2 algorithm with a SHA256 hash, a password stretching mechanism recommended by NIST", which is deemed very secure, requiring "massive amounts of computing time to break". Django adds a salt to the password, along with the number of iterations of the SHA256 hash that are performed.

As a result, the security of passwords stored in the database is not a general concern in Django (provided the overall security of the application is enforced).

Django additionally makes sure to enforce password policies by default to ensure that a user doesn't provide weak credentials, which can be seen by viewing the register page below. If the user does not conform to the policies, an error will be thrown, and the user will not be able to access the application.

Additionally, in Django, it is very easy to ensure that a user can only access a site if he is authenticated. All that is required is to use Django's in-build library, "django.contrib.auth", which also contains Django's built-in login and logout modules, streamlining that process as well.
