The Admin Page

A common vulnerability in Django applications is the admin page, as the endpoint, "/admin", is very-well-known.

  • Default admin pages (like /admin) are common targets for hackers and bots.
  • By changing the URL to these pages, you can reduce the chances of automated attacks and brute force attempts by bots that might discover the page..
  • I also added a honeypot on the real admin page to track if there are any threat actors on the site and to lure bots. (Doesn't actually work!)
  • I added captcha to honeypot page in case a robot manages to find the real admin page
  • Access control ensures that each user can only access files (documents, images, etc.) related to their role or work.
